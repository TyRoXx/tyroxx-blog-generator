How to choose an integer type

std ::uint32_t over the types without an explicit range. Use int, long, long longonly if you have a reason to use exactly these and not the portable ones. C++ is different from Java and C#: int etc are not the same size and range on every platform. This makes them hard to use even for C++ experts. They are neither necessary nor sufficient for most practical situations. In most cases you already know the expected range of values. So why not use the right integer type for this instead instead of cargo-culting it? If you know that you are dealing with 64-bit file sizes, document this assumption in an unambiguous way. There is no reason for using int or long long in this case. Using the wrong type leads to unportable code and misunderstandings.
